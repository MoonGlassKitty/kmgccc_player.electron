# mgkccc_player 使用说明

这份说明写给第一次使用 mgkccc_player 的人。软件目前还有一些实验性功能，界面里不一定会主动解释每个操作，所以这里把常用入口和隐藏操作写清楚。

## 启动和基础播放

1. 打开 `mgkccc_player.exe`。
2. 左侧栏选择播放源：
   - `本地`：播放软件内导入的本地音乐。
   - `第三方音乐软件` / `其他源` / `自动检测`：读取 Windows 正在播放的外部媒体状态。
3. 在主页、所有歌曲、播放列表、艺人页或专辑页中点击歌曲即可播放。
4. 底部播放器可以控制播放、暂停、上一首、下一首、音量、播放模式和播放列表。

第三方播放源依赖 Windows 的系统媒体控制信息。如果外部播放器没有把标题、封面、进度或控制能力暴露给系统，mgkccc_player 里对应内容会缺失，部分按钮也会变灰。

## 左侧栏

- 左上角按钮：收起或展开侧栏。
- 播放列表区域右侧的 `+`：新建播放列表。
- 艺人、专辑标题行：点击可展开。
- 底部三个圆形按钮：
  - 齿轮：打开设置。
  - 太阳/月亮：切换浅色或深色。
  - 全屏图标：进入全屏歌词/全屏播放界面。

如果侧栏内容很多，可以直接滚动侧栏区域。

## 播放列表和队列

底部播放器右侧的播放列表按钮可以打开当前播放队列。

- 点击队列里的歌曲：切换播放。
- 拖动队列里的歌曲：调整播放顺序。
- 点击歌曲右侧删除按钮：从当前队列移除。
- 专辑页里播放按钮和编辑按钮之间的添加按钮：把整张专辑加入当前播放队列。

播放模式按钮会循环切换：

- 随机播放
- 单曲循环播放
- 列表循环播放
- 列表单次播放

第三方播放源暂时不能可靠读取外部播放器的播放模式，所以播放模式按钮可能会变灰。

## 歌词

右侧歌词栏显示当前歌曲歌词。底部工具栏的歌词按钮可以打开或关闭歌词侧栏。

没有歌词时会显示：

> 纯音乐，你的故事在这里。

歌词相关设置在 `设置 -> 窗口播放 -> 歌词` 和 `设置 -> 全屏播放 -> 歌词` 中。

常用设置：

- 字体大小
- 翻译大小
- 浅色/深色模式字重
- 中文字体、英文字体、翻译字体
- 歌词渲染质量
- 歌词整体提前量

如果第三方播放源的歌词明显慢或快，优先调 `歌词整体提前量`。正值会让歌词更早显示，负值会让歌词更晚显示。

## 全屏歌词

点击左下角全屏按钮进入全屏歌词界面。

全屏歌词皮肤在 `设置 -> 全屏播放 -> 皮肤` 中选择。部分皮肤会根据封面取色，切换歌曲或皮肤时颜色会重新计算。

全屏歌词如果没有显示歌词，先确认：

1. 当前歌曲是否有歌词。
2. 歌词侧栏/全屏歌词是否被隐藏。
3. 第三方播放源是否成功读取到当前歌曲信息。

## 窗口播放皮肤

设置路径：`设置 -> 窗口播放 -> 常规`。

常见皮肤：

- 封面 LED：经典封面和节拍律动相关功能主要在这个皮肤里使用。
- Apple 风格：封面渐变、柔和背景。
- 旋转封面：唱片或封面旋转效果。
- 磁带风格：磁带外观和 LED 电平表。

如果设备性能不足，可以关闭 `启用艺术背景` 或把歌词渲染质量调低。

## 节拍器/封面节拍律动

节拍律动主要用于 `窗口播放 -> 封面 LED` 皮肤。这个功能的入口不明显，需要按下面步骤操作。

### 开启或关闭节拍律动

1. 进入 `设置 -> 窗口播放 -> 常规`。
2. 选择 `封面 LED` 皮肤。
3. 回到窗口播放界面。
4. 单击正在播放的封面。
5. 封面会按检测到的 BPM 进行缩放律动；再次单击可关闭。

### 手动输入 BPM

1. 确认当前是 `封面 LED` 皮肤。
2. 在窗口播放界面双击封面。
3. 打开手动 BPM 输入窗口后，输入 BPM。
4. 当前允许的最低 BPM 是 `25`。
5. 保存后，封面会按这个 BPM 打节拍。

适合用于自动检测不准、纯音乐、第三方播放源识别不稳定的情况。

### 保存当前自动检测节拍

1. 播放歌曲并打开节拍律动。
2. 等待软件自动检测 BPM，封面开始跟随节奏。
3. 长按封面约半秒以上。
4. 出现 BPM 提示后，当前歌曲的节拍会被记住。

下次播放同一首歌时，会优先使用保存过的节拍。

### 节拍打不起来时

按这个顺序排查：

1. 当前皮肤是否是 `封面 LED`。
2. 节拍律动是否已经通过单击封面开启。
3. 本地歌曲是否有可读取的音频文件。
4. 第三方播放源是否只给了封面和标题，没有给可分析的音频地址。
5. 如果自动检测失败，双击封面手动输入 BPM。

第三方源不一定能提供真实音频文件，软件可能只能读到标题、艺人、封面和进度。这种情况下自动分析 BPM 可能失败，手动 BPM 更可靠。

## 第三方播放源

左侧底部播放源可以切到第三方：

- `第三方音乐软件`：优先读取常见音乐软件。
- `其他源`：读取其他系统媒体源。
- `自动检测`：由软件尝试选择可用来源。

第三方源可用时，mgkccc_player 会尽量读取：

- 歌曲名
- 艺人
- 封面
- 播放/暂停状态
- 进度
- 歌词

限制：

- 外部软件没有暴露的信息，mgkccc_player 不能凭空获取。
- 有些外部软件不支持远程上一首/下一首/播放暂停，对应按钮会变灰或无效。
- 第三方源歌词可能有延迟，可以在歌词设置里调整提前量。

## 设置里的常用项目

### 外观

路径：`设置 -> 外观`

- 深色/浅色跟随系统
- 手动外观
- 歌词卡片背景
- 网页大小缩放

网页大小缩放相当于浏览器里的 `Ctrl + +` / `Ctrl + -`，范围是 `50%` 到 `125%`，默认 `80%`。

### 窗口播放

路径：`设置 -> 窗口播放`

- 皮肤选择
- 艺术背景
- LED/频谱动画
- 歌词字体、字重、时间补偿

### 全屏播放

路径：`设置 -> 全屏播放`

- 全屏皮肤
- 全屏歌词字体大小
- 全屏歌词翻译大小
- 全屏歌词整体提前量

### 音频

路径：`设置 -> 音频`

主要用于第三方播放源和歌词同步的延迟补偿。

### 数据

路径：`设置 -> 数据`

用于初始化或清理本地数据。使用前确认不再需要当前设置、播放状态或本地缓存。

## 常见问题

### 为什么第三方源有些按钮不能点？

外部播放器没有向 Windows 暴露控制能力，或者当前来源只允许读取状态，不允许控制播放。

### 为什么某首歌没有歌词？

本地元数据没有歌词，在线匹配也没有找到结果时，会显示纯音乐提示。可以在歌曲元数据编辑里手动补歌词。

### 为什么封面节拍和歌曲不同步？

可能是自动 BPM 检测不准，或当前歌曲前奏/空拍较长。双击封面手动输入 BPM，或者长按保存一个自动检测到的合适 BPM。

### 为什么切到第三方源后节拍打不了？

第三方源通常不给真实音频文件，只给系统媒体状态。没有音频文件时软件无法分析 BPM，需要手动 BPM。

### 为什么打包后的 exe 图标不是默认 Electron 图标了？

现在打包配置已经使用 `windows-player/build/icon.ico`，窗口运行时也会读取同一套图标资源。
