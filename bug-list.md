```text
BUG 编号：BUG-0002
BUG 名字：全屏播放模式Dock鼠标不悬停后自动隐藏延迟和分块悬停判定不同步
BUG 记录时间：2026-06-14
发生的系统：Windows
BUG 的大致类型：界面 / 播放
复现方法：进入全屏播放后，如果鼠标不悬停在 Dock，Dock 会延迟一段时间后消失，但当前消失延迟偏短，需要加长延迟时间，或增加设置选项让用户更改延迟消失时间。底部 Dock 被分为两块，鼠标悬停在某一块时另一块会消失，应该改成两块同步消失或同步出现。鼠标悬停在横贯底部的共同判定区域时，两块 Dock 会同时出现和消失；但每一块 Dock 又有自己的悬停判定区域，并且叠在共同的横贯判定区域上，导致鼠标悬停到其中一块 Dock 后另一块 Dock 消失。
BUG 的状态：已复现
```

```text
BUG 编号：BUG-0005
BUG 名字：dock的播放列表鼠标移开后不消失,包括左侧其他源展开的列表,以及其他所有可能的列表
BUG 记录时间：2026-06-14
发生的系统：Windows
BUG 的大致类型：界面 / 播放
复现方法：1.非全屏播放,点击dock里的播放列表,播放列表弹出后一直存在,直到再次点击播放列表按钮,应当优化为点击播放列表按钮后播放列表弹出,同时在播放列表的区域增加一个悬停用的判定范围,这个判定范围的底部应该延长至播放列表按钮,即点击播放列表按钮后鼠标位于判定范围内,防止播放列表消失,鼠标移出判定范围的时候播放列表自动消失(可选添加动画)2.全屏播放的时候,播放列表弹出后会随着dock一起消失或者出现,在添加了之前的消失逻辑以后需要先让播放列表消失再让dock消失,播放列表的消失判定逻辑应该优先处理,即全屏模式下,如果鼠标先悬停在dock然后点击播放列表,然后鼠标移动到播放列表,此时dock不应该消失,dock的消失应该把播放列表消失作为前置条件,对于以后需要添加在dock的其他功能案件也需要遵循该逻辑.
BUG 的状态：已复现
```

### 状态说明

- 待复现：发现 BUG，但是目前还不知道如何稳定复现。
- 已复现：已经可以稳定复现，但是还没有修复。
- 已修复：已经完成修复。

### 记录规则

- BUG 记录需要有序号，例如 `BUG-0001`、`BUG-0002`。
- BUG 记录时间使用发现或记录该 BUG 的日期，格式为 `YYYY-MM-DD`。
